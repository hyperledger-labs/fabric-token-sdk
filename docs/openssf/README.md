# OpenSSF Best Practices Assessment

This directory contains the self-assessment documents for the **OpenSSF Best Practices Badge Program**.

## What is OpenSSF?
The [Open Source Security Foundation (OpenSSF)](https://openssf.org/) is a cross-industry organization hosted by the Linux Foundation. Its mission is to collaborate on improving the security of open-source software (OSS).

One of its key initiatives is the **Best Practices Badge Program**, which provides a set of criteria for open-source projects to show that they follow security and development best practices.

## Assessment Levels
The assessment is divided into three "Baseline Levels," representing increasing maturity:

1.  [**Level 1 (Passing)**](./openssf_baseline_level_1.md): Covers basic project hygiene, documentation, and security essentials.
2.  [**Level 2 (Silver)**](./openssf_baseline_level_2.md): Focuses on security architecture, vulnerability management, and enhanced governance.
3.  [**Level 3 (Gold)**](./openssf_baseline_level_3.md): Requires advanced controls, SBOMs, and formal threat modeling.

## How to Reassess
The documents in this directory serve as a point-in-time self-assessment. To reassess the project's compliance:

1.  Visit the links above to see the latest criteria and current project status on the OpenSSF website.
2.  Review the `Assessment` and `Notes` columns in the corresponding `.md` files.
3.  Identify any "Not Met" or "?" items and determine if the project has since implemented those practices.
4.  Update the markdown files with new evidence or assessment results.
5.  If significant progress is made, consider updating the project's official badge status on the [Best Practices website](https://www.bestpractices.dev/en/projects/7176).
